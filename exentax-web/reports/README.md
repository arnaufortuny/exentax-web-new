# `exentax-web/reports/`

Audit deliverables. Most files in this directory are gitignored
(re-generated on every audit run); the ones tracked in git are
listed by name in `exentax-web/.gitignore` under the `!reports/...`
exceptions and serve as the durable record of each audit.

## Task #19 — CTA / WhatsApp / navigation audit

The full write-up lives in **`cta-audit-report.md`** (start here).
The supporting datasets fall into two layers:

### Canonical merged inventory (rendered DOM ⨝ static source)

- **`cta-canonical.csv` / `.json`** — one row per rendered CTA × locale
  with `page,lang,surface,category,href,text,testid,matched_by,
  file_lines,source_labels,tracking`. Built by
  `scripts/cta-canonical.mjs`, which joins the rendered DOM crawl
  below with the static source scan along **two** axes:

  1. exact `data-testid` match, and
  2. resolved-URL match — every static row whose destination is a
     route key (e.g. `lp("book")`) is expanded via
     `ROUTE_SLUGS` to all 6 per-locale URLs and indexed under each.

  Use this when you want a single artifact that answers *"for this
  CTA, what page/locale is it on, what does it say, where does it
  go, where is it defined in the source, and what tracking event
  does it fire?"*. The `matched_by` column on every row records
  *how* the join was made (`testid` / `url` / blank); `_meta`
  records the join rate breakdown for the run, plus a
  `column_completeness` map that spells out which columns are
  guaranteed-complete (`page,lang,surface,category,href,text`) vs
  best-effort because they depend on the regex-based static parser
  (`file_lines,source_labels,tracking`). Re-derive at any time
  with `node scripts/cta-canonical.mjs` (no browser, no dev
  server — it just re-joins the two existing JSON files).

### Authoritative — what visitors actually see (Playwright DOM crawl)

- **`cta-rendered-inventory.csv` / `.json`** — every `<a href>`
  rendered on the 6 audited surfaces × 6 locales after full
  client-side hydration (i18n + LangSyncEffect/BlogLangEffect).
  Each row carries `page,lang,surface,category,href,text,testid`,
  giving you in a single artifact the {destination, visible copy,
  tracking-event id} triple for every CTA in every language —
  no JOIN against the static inventory required.
  Use this dataset when you need the truth about what CTAs each
  visitor in each language is exposed to. Generated by
  `scripts/cta-rendered-crawler.mjs`.
- `cta-rendered-crawler.md` / `.json` — per-surface summary +
  pass/fail verdict (broken count, wa.me number consistency).

### Companion — what the source code declares (static regex scan)

- `cta-inventory.{md,csv,json}` — every `<Link>` / `<a>` /
  `wa.me` / `mailto:` site found by walking
  `client/src/**/*.{ts,tsx}`. Each row carries the originating
  `file:line` so reviewers can jump straight into the code.
- `cta-inventory-by-locale.{csv,json}` — the same 73 source-code
  rows expanded to 438 rows (× 6 languages) with the resolved
  localized URL and the resolved localized copy.

The static inventory is a useful code-review and code-search aid,
but it cannot see CTAs whose `href`, `label`, or `data-testid`
is supplied through a variable, prop, or `t()` lookup; for those,
defer to the rendered inventory above. A small number of rows in
the static CSVs may also appear with partial / templated string
fragments — by design, since they reflect the source expression
rather than a runtime evaluation.

### Specialist datasets

- `whatsapp-inventory.{csv,json}` + `whatsapp-audit.md` — every
  `wa.me` link grouped by phone number, with the canonical number
  pulled from `client/src/lib/constants.ts`. Generated by
  `scripts/cta-whatsapp-audit.mjs`.
- `cta-crawler.{md,json}` — registered-route consistency
  (`ROUTE_SLUGS` × 6 locales) plus a live HTTP smoke check
  against the dev server. Generated by
  `scripts/cta-link-crawler.mjs`.

## Re-running

See **§5 of `cta-audit-report.md`** for the full command list.
TL;DR (from `exentax-web/`):

```bash
node scripts/cta-inventory.mjs
node scripts/cta-whatsapp-audit.mjs
node scripts/cta-link-crawler.mjs           # needs dev server on :5000
node scripts/cta-rendered-crawler.mjs        # needs dev server on :5000 + chromium
npx playwright test tests/e2e/cta-navigation.spec.ts
```
