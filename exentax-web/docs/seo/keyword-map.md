# Exentax — Multilingual keyword map 2026

Six-language keyword strategy. Primary language is Spanish; the other five follow the same cluster structure with localized intent modifiers.

Legend:
- **KW** = target keyword (exact)
- **Intent** = I (informational) · C (commercial) · T (transactional) · N (navigational)
- **Vol** = rough monthly search volume in the primary market (Spain for `es`, UK/US for `en`, France for `fr`, DACH for `de`, Portugal for `pt`, Catalonia for `ca`). Based on aggregated third-party data and SERP competitiveness, not direct measurements.
- **Diff** = keyword difficulty 0-100 (rough estimate).
- **URL** = canonical target page on exentax.com.

## 9. SERP features to target

- **Featured snippet** (paragraph): define-style answers in `<p>` of 40–60 words at the top of each pillar.
- **People Also Ask**: dedicated FAQ block at the bottom with H3 questions.
- **Jump-links**: table of contents with anchor links at the start of pillars.
- **Sitelinks search box**: add `WebSite` schema with `SearchAction` on home (needs a working `/es/blog?q=` search endpoint — product decision).

## 10. Keyword cannibalization guardrails

When any new post is drafted:

1. Search `client/src/data/blog-posts.ts` and `blog-posts-i18n.ts` for the primary KW; if an existing post ranks or targets it, **merge or canonicalize**, do not create a duplicate.
2. Update `publishing-checklist.md` to enforce this.
